# site-design-vanilla-js
Site design for learning purposes.

Вимоги (functional / non-functional)
Functional requirements

1. Показувати список товарів (картки): назва, категорія, рейтинг, ціна, статус “в наявності”, кнопка “Додати в кошик”.
2. Пошук товарів за назвою (input + live filtering).
3. Фільтрація:
- за категорією (checkbox)
- за “В наявності” (toggle)
- за діапазоном цін (min/max)
4. Сортування: за ціною (зростання/спадання), за рейтингом, за назвою.
5. Кнопка “Скинути фільтри”.
6. Кошик:
- відкриття/закриття модального вікна
- додавання/видалення позицій
- зміна кількості (+ / −)
- підрахунок subtotal
- badge з кількістю товарів у кошику
7. Повідомлення “Нічого не знайдено” якщо після фільтрації/пошуку немає товарів.

Non-functional requirements
1. Семантичний HTML: header, main, section, aside, article, nav, footer.
2. Доступність: видимі :focus, aria-атрибути для модалки, кнопок, лейблів.
3. Адаптивність: нормально на ширині 360px+ (flex/grid).
4. CSS в окремому файлі і підключений через <link>.
5. Код читабельний: класи по BEM-подібній схемі, без інлайн-стилів.
6. JS події реально впливають на функціональність (фільтрація, сортування, кошик, модалка).



Дизайн (schema) сторінки

Header
- Лого/назва
- Рядок пошуку
- Кнопка “Кошик” + badge

Main
- aside (ліва панель): фільтри (категорії, наявність, ціна, сортування, reset)
- section (права частина):
- панель результатів (кількість знайдених)
- grid з картками товарів

Modal
- Список позицій кошика з керуванням кількістю
- Subtotal
- Кнопка “Очистити”

Footer
- Коротка довідка
