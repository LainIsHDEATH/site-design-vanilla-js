# site-design-vanilla-js
Site design for learning purposes.

Вимоги (functional / non-functional)
Functional requirements

Показувати список товарів (картки): назва, категорія, рейтинг, ціна, статус “в наявності”, кнопка “Додати в кошик”.

Пошук товарів за назвою (input + live filtering).

Фільтрація:

за категорією (checkbox)

за “В наявності” (toggle)

за діапазоном цін (min/max)

Сортування: за ціною (зростання/спадання), за рейтингом, за назвою.

Кнопка “Скинути фільтри”.

Кошик:

відкриття/закриття модального вікна

додавання/видалення позицій

зміна кількості (+ / −)

підрахунок subtotal

badge з кількістю товарів у кошику

Повідомлення “Нічого не знайдено” якщо після фільтрації/пошуку немає товарів.

Non-functional requirements

Семантичний HTML: header, main, section, aside, article, nav, footer.

Доступність: видимі :focus, aria-атрибути для модалки, кнопок, лейблів.

Адаптивність: нормально на ширині 360px+ (flex/grid).

CSS в окремому файлі і підключений через <link>.

Код читабельний: класи по BEM-подібній схемі, без інлайн-стилів.

JS події реально впливають на функціональність (фільтрація, сортування, кошик, модалка).
